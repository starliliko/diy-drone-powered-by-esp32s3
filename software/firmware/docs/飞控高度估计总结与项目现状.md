# 飞控高度估计总结与项目现状（精简版）

本文档只保留与当前工程实现直接相关的高度估计内容，删除旧版本里重复理论、过时频率与旧链路描述。

配套详解见：文档/卡尔曼估计器详解.md

## 1. 当前项目的高度估计结论

一句话：

- 当前默认是 Kalman 估计器
- 高度主观测优先 TOF
- Baro 驱动与融合代码都在，但默认编译不开启 Baro 更新

因此当前系统可概括为：

- IMU 高频预测 + TOF 近地强约束
- Baro 保留为可选路径

## 2. 当前有效数据链路

### 2.1 预测链（高频）

- 来源：BMI088 加速度 + 陀螺仪
- 作用：连续传播 z 与 vz（以及姿态相关状态）
- 频率：Kalman 预测 500Hz

说明：

- 代码仍传入 thrustAverage，但当前预测方程本身未直接使用 thrust 做动力学传播
- thrust 主要用于 quadIsFlying 判定

### 2.2 TOF 更新链（主观测）

- 来源：MTF01 距离
- 作用：更新高度状态 Z
- 关键保护：
  - 首次有效值初始化高度
  - 倾斜补偿（基于 R[2][2]）
  - innovation > 0.3m 增噪
  - innovation > 1.0m 直接重置高度

### 2.3 Baro 更新链（可选）

- 来源：MS5611（压力换算 asl）
- 状态：代码存在，默认关闭
- 开启条件：定义 KALMAN_USE_BARO_UPDATE
- 开启后行为：
  - 50 样本参考高度校准
  - 使用相对高度更新 Z

## 3. 当前实现里容易误解的点

1. 不是“Kalman 任务 100Hz”
   - 当前是 500Hz 预测 + 25Hz 加速度姿态校正 + 50Hz Baro 条件更新

2. 不是“MTF01 按旧 dpixel 光流更新”
   - 当前是 MTF01 在驱动侧处理后，直接作为速度量测更新 PX/PY

3. 不是“Baro 已彻底移除”
   - 驱动在、换算在、Kalman 更新函数在
   - 只是默认编译开关未打开

## 4. 调试时建议优先观察的量

建议先看 Kalman 日志组中的：

- 高度与速度：stateZ, statePZ
- 对应不确定度：varZ, varPZ
- 频率计数：rtPred, rtBaro, rtUpdate, rtFinal
- 量测队列健康度：rtApnd, rtRej

快速判读：

- rtPred 接近 500 且 rtBaro 接近 0
  - 说明系统符合“TOF 主导、Baro 默认关闭”的当前配置

## 5. 现阶段可执行优化顺序

1. 保持 TOF 主链稳定
   - 先确认近地高度与刹车工况稳定性

2. 完善 TOF 置信度逻辑
   - 新鲜度、量程边界、表面异常检测

3. 再引入 Baro 低频约束
   - 先开编译开关，再保守调 mNBaro

4. 最后联动控制器调参
   - 在估计稳定后再做定高控制增益优化

## 6. 当前版本总结

当前工程已经具备可用的近地高度估计主链：

- IMU 连续传播
- TOF 高权重更新
- Baro 可扩展接入

这是一套偏室内、偏近地、工程保护较强的实现。

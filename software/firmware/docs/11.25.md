## 上周遗留问题
### IMU数据处理的流程

#### 1.陀螺仪校准过程

由于陀螺仪存在较大的误差，包括零漂和温漂，因此每次使用前需要对陀螺仪进行校准，计算当前环境下的陀螺仪基准值：**在初次上电时，计算陀螺仪三个轴的方差与平均值。**

陀螺仪具体校准过程如下：

使用一个最大长度为 1024 的环形缓冲区，存储最新的 1024 组陀螺仪测量值。

通过计算陀螺仪输出值方差，确认飞行器已经放置平稳并且陀螺仪工作正常。

确认第 2 步正常后，计算静止时 1024 组陀螺仪输出值的平均值，作为陀螺仪的校准值。

代码如下：

```c
static void sensorsAddBiasValue(BiasObj *bias, int16_t x, int16_t y, int16_t z)
{
    bias->bufHead->x = x;
    bias->bufHead->y = y;
    bias->bufHead->z = z;
    bias->bufHead++;

    if (bias->bufHead >= &bias->buffer[SENSORS_NBR_OF_BIAS_SAMPLES])
    {
        bias->bufHead = bias->buffer;
        bias->isBufferFilled = true;
    }
}

static bool sensorsFindBiasValue(BiasObj *bias)
{
    static int32_t varianceSampleTime;
    bool foundBias = false;

    if (bias->isBufferFilled)
    {
        sensorsCalculateVarianceAndMean(bias, &bias->variance, &bias->mean);

        if (bias->variance.x < GYRO_VARIANCE_THRESHOLD_X &&
            bias->variance.y < GYRO_VARIANCE_THRESHOLD_Y &&
            bias->variance.z < GYRO_VARIANCE_THRESHOLD_Z &&
            (varianceSampleTime + GYRO_MIN_BIAS_TIMEOUT_MS < xTaskGetTickCount()))
        {
            varianceSampleTime = xTaskGetTickCount();
            bias->bias.x = bias->mean.x;
            bias->bias.y = bias->mean.y;
            bias->bias.z = bias->mean.z;
            foundBias = true;
            bias->isBiasValueFound = true;
        }
    }

    return foundBias;
}
```

#### 加速度计校准过程
做重力加速度校准

在地球不同的纬度和海拔下，重力加速度 g 值一般不同，因此需要使用加速度计对 g 进行实际测量。
g 值的校准过程如下：

陀螺仪校准完成后，立刻进行加速度计校准。

使用 Buffer 保存 200 组加速度计测量值。

通过合成重力加速度在三个轴的分量，计算重力加速度在静止状态下的值。

计算静止状态下重力加速度值

代码如下:

```c
static bool processAccScale(int16_t ax, int16_t ay, int16_t az)
{
    if (!accScaleFound)
    {
        accScaleSum += sqrtf(powf(ax * SENSORS_BMI088_G_PER_LSB_CFG, 2) +
                             powf(ay * SENSORS_BMI088_G_PER_LSB_CFG, 2) +
                             powf(az * SENSORS_BMI088_G_PER_LSB_CFG, 2));
        accScaleSumCount++;

        if (accScaleSumCount == SENSORS_ACC_SCALE_SAMPLES)
        {
            accScale = accScaleSum / SENSORS_ACC_SCALE_SAMPLES;
            accScaleFound = true;
        }
    }

    return accScaleFound;
}
```

#### 滤波器处理输出数据
在经过陀螺仪校准和加速度计校准后，加入修正，再通过低通滤波器过滤高频噪音，最后作为输入控制系统的数据

代码如下:

```c
// 陀螺仪数据处理
sensorData.gyro.x = (gyroRaw.x - gyroBias.x) * SENSORS_BMI088_DEG_PER_LSB_CFG;
sensorData.gyro.y = (gyroRaw.y - gyroBias.y) * SENSORS_BMI088_DEG_PER_LSB_CFG;
sensorData.gyro.z = (gyroRaw.z - gyroBias.z) * SENSORS_BMI088_DEG_PER_LSB_CFG;
applyAxis3fLpf((lpf2pData *)(&gyroLpf), &sensorData.gyro);

// 加速度计数据处理
accScaled.x = accelRaw.x * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
accScaled.y = accelRaw.y * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
accScaled.z = accelRaw.z * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
sensorsAccAlignToGravity(&accScaled, &sensorData.acc);
applyAxis3fLpf((lpf2pData *)(&accLpf), &sensorData.acc);
```

其中低通滤波函数 `applyAxis3fLpf` 的实现：

```c
static void applyAxis3fLpf(lpf2pData *data, Axis3f *in)
{
    for (uint8_t i = 0; i < 3; i++)
    {
        in->axis[i] = lpf2pApply(&data[i], in->axis[i]);
    }
}
```

#### 机身水平校准
理想状态下，加速度传感器在小飞机上完全水平地进行安装，进而可以使用 0 位置作为小飞机的水平面。但由于加速度计在安装时不可避免的存在一定的倾角，导致飞控系统错误估计水平位置，导致小飞机向某个方向偏飞。因此需要设置一定的校准策略来平衡这种误差。

将小飞机放置在一个水平面上，计算小飞机 cosRoll、sinRoll、cosPitch、sinPitch。理想状态下 cosRoll、cosPitch 为 1，sinPitch、sinRoll 为 0。如果不是水平安装 sinPitch、sinRoll 不为 0，cosRoll cosPitch 不为 1。

将步骤 1 的 cosRoll、sinRoll、cosPitch、sinPitch 或对应的 Roll、Pitch 角度值保存到飞机，用于校准。

利用校准值，对加速度计测量值进行修正：

代码如下：

```c
static void sensorsAccAlignToGravity(Axis3f *in, Axis3f *out)
{
    Axis3f rx;
    Axis3f ry;

    // Rotate around x-axis
    rx.x = in->x;
    rx.y = in->y * cosRoll - in->z * sinRoll;
    rx.z = in->y * sinRoll + in->z * cosRoll;

    // Rotate around y-axis
    ry.x = rx.x * cosPitch - rx.z * sinPitch;
    ry.y = rx.y;
    ry.z = -rx.x * sinPitch + rx.z * cosPitch;

    out->x = ry.x;
    out->y = ry.y;
    out->z = ry.z;
}
```

## 本周工作
### 飞控系统的主要任务
Crazyflie 是基于 FreeRTOS，所有 FreeRTOS 的任务设置可在 **components\config\include\config.h** 文件修改

#### 任务优先级分配（数字越大优先级越高）

系统按功能模块划分为以下几类任务，优先级从低到高：

##### 1. 系统状态管理任务（优先级 1）
- **SYSTEM_TASK** (优先级 1)：系统初始化与状态管理
- **PM_TASK** (优先级 1)：电源管理任务 实际用不上
- **LEDSEQCMD_TASK** (优先级 1)：LED 灯控制 实际是控制led灯闪烁表示信息
- **KALMAN_TASK** (优先级 4，单核时降为 1)：卡尔曼滤波状态估计 重要

##### 2. 通信任务（优先级 2）
**发送任务 (TX)：**
- **UDP_TX_TASK** (优先级 2)：UDP 数据发送
- **CRTP_TX_TASK** (优先级 2)：CRTP 协议数据发送

**接收任务 (RX)：**
- **UDP_RX_TASK** (优先级 2)：UDP 数据接收
- **EXTRX_TASK** (优先级 2)：外部接收器任务
- **UART2_TASK** (优先级 2)：UART2 串口通信 目前没用，之后考虑接入SUBS协议兼容实物遥控器
- **SYSLINK_TASK** (优先级 2)：系统链路通信 目前没用，被WiFi代替了
- **USBLINK_TASK** (优先级 2)：USB 链路通信 没用
- **WIFILINK_TASK** (优先级 2)：Wi-Fi 链路通信 重要,通信的底层实现
- **CRTP_RX_TASK** (优先级 2)：CRTP 协议数据接收

**数据处理任务 (优先级 2-3)：**
- **INFO_TASK** (优先级 2)：信息管理 没有用
- **LOG_TASK** (优先级 2)：日志记录 
- **MEM_TASK** (优先级 2)：内存管理
- **PARAM_TASK** (优先级 2)：参数管理
- **CMD_HIGH_LEVEL_TASK** (优先级 3)：高层命令处理

##### 3. 传感器与稳定控制任务（优先级 4-7，最高优先级）
- **PROXIMITY_TASK** (优先级 5)：距离传感器任务 没有用
- **FLOW_TASK** (优先级 5)：光流传感器任务
- **ZRANGER2_TASK** (优先级 5)：Z-Ranger2 测距任务
- **ZRANGER_TASK** (优先级 5)：Z-Ranger 测距任务
- **SENSORS_TASK** (优先级 6)：传感器数据采集与处理
- **STABILIZER_TASK** (优先级 7)：**最高优先级**，飞控核心稳定控制循环

#### 核心任务功能分析

##### STABILIZER_TASK（优先级 7，最高）
- **功能**：1000Hz 飞控核心循环，负责读取传感器 → 状态估计 → PID 控制 → 电机输出
- **重要性**：整个飞控系统的心脏，必须以最高优先级运行保证实时性
- **堆栈**：5× 基准堆栈，用于控制算法计算

##### SENSORS_TASK（优先级 6）
- **功能**：从 IMU、气压计等传感器读取原始数据，进行校准、滤波处理
- **数据流**：硬件中断 → 读取原始数据 → 校准 → 低通滤波 → 传递给 STABILIZER_TASK
- **堆栈**：5× 基准堆栈，用于传感器数据处理与滤波

##### KALMAN_TASK（优先级 4，单核降为 1）
- **功能**：卡尔曼滤波器状态估计，融合多传感器数据（IMU、气压计、光流等）
- **特殊处理**：在单核系统中优先级降为 1，因为卡尔曼滤波计算量大，避免阻塞其他关键任务
- **堆栈**：3× 基准堆栈

##### 通信任务（优先级 2）
- **WIFILINK_TASK / UDP_RX_TASK / UDP_TX_TASK**：处理 Wi-Fi/UDP 通信，接收控制指令、发送遥测数据
- **CRTP_RX_TASK / CRTP_TX_TASK**：Crazy RealTime Protocol，与 Crazyflie 客户端通信的核心协议
- **堆栈分配**：通信任务分配较大堆栈（3-4×）用于网络缓冲区

##### 系统管理任务（优先级 1-2）
- **PARAM_TASK / LOG_TASK**：参数调整与日志记录，支持运行时参数调优和遥测
- **PM_TASK**：电池电压监控、低电压保护

#### 优先级设计原则总结

1. **实时控制最高优先级**（6-7）：STABILIZER 和 SENSORS 确保 1000Hz 控制循环不被打断
2. **传感器融合次高优先级**（4-5）：光流、测距等辅助传感器
3. **通信与数据处理中等优先级**（2-3）：保证指令及时响应，但不影响飞控稳定
4. **系统管理最低优先级**（1）：LED、电源管理等非实时任务

**注意事项**：
- 单核系统（`CONFIG_FREERTOS_UNICORE`）会将卡尔曼任务优先级降为 1，避免 CPU 占用过高
- FreeRTOS 配置要求 `CONFIG_FREERTOS_HZ=1000`（1ms tick）以支持精确的 1000Hz 控制

# 卡尔曼估计器详解（按当前代码）

本文档描述当前工程中卡尔曼估计器的真实实现，以源码为准。
目标是替换旧版中已经过时的内容（旧任务频率、旧光流更新模型、旧文件职责划分）。

## 1. 文档范围与结论

当前估计器是误差状态 EKF（ESKF）风格实现，采用“前台快速采集 + 后台独立融合”架构：

- 稳定器主循环在约 1kHz 节拍中调用 estimator 入口
- 独立 Kalman 任务执行预测/更新/Finalize/导出
- 预测主频 500Hz
- 加速度姿态校正 25Hz
- 气压计更新代码存在，但默认由编译开关控制，当前默认不启用
- TOF 和 MTF01 速度量测是当前常用观测输入

## 2. 核心文件（当前职责）

- components/core/crazyflie/modules/src/estimator_kalman_task.c
  - Kalman 任务调度主流程（频率节拍、预测、更新、Finalize、状态导出）
- components/core/crazyflie/modules/src/kalman_core.c
  - 核心滤波数学实现（状态传播、标量更新、各类量测模型）
- components/core/crazyflie/modules/src/estimator_kalman_buffer.c
  - IMU/推力/Baro 累计平均缓冲
- components/core/crazyflie/modules/src/estimator_kalman_measurements.c
  - 异步量测队列（TOF/速度/位置/位姿/TDOA 等）
- components/core/crazyflie/modules/src/estimator_kalman_sync.c
  - 互斥锁与运行信号量
- components/core/crazyflie/modules/src/estimator_kalman.c
  - 对外状态、日志、参数注册

## 3. 系统启动与调度链路

1. 系统默认选择 kalmanEstimator，并启动 Kalman 任务。
2. stabilizer 主循环每次传感器 ready 后调用 stateEstimator。
3. 当前 stateEstimator 会转发到 estimatorKalman（Kalman 模式）。
4. estimatorKalman 做“快路径”：
   - 读取传感器
   - 累计到 buffer
   - 返回上一轮已导出的 taskEstimatorState
   - 发信号唤醒后台 Kalman 任务
5. 后台 Kalman 任务执行完整融合并更新 taskEstimatorState。

这个设计让控制主循环保持低延迟，不被 EKF 计算阻塞。

## 4. 状态定义与坐标系

内部状态向量 S[9]：

- S[X,Y,Z]：世界系位置
- S[PX,PY,PZ]：机体系速度
- S[D0,D1,D2]：姿态误差小量

附加状态：

- q[4]：姿态四元数（主姿态）
- R[3][3]：旋转矩阵（机体到世界）
- P[9][9]：协方差矩阵

导出到 state_t 时：

- 位置直接输出世界系
- 速度由 R * [PX,PY,PZ] 转到世界系
- 欧拉角由 q 转换，pitch 保留历史兼容符号处理

## 5. 当前真实执行频率

在 estimator_kalman_task.c 中固定为：

- PREDICT_RATE = 500Hz
- BARO_RATE = 50Hz（仅在开启 Baro 更新宏时有效）
- ACCEL_ATTITUDE_RATE = 25Hz

说明：旧文档中的“Kalman 任务 100Hz”不再准确。

## 6. 主循环执行顺序（后台 Kalman 任务）

每次被 runTaskSemaphore 唤醒后：

1. 可选重置
   - 如果 resetEstimation 置位，重新初始化核心状态
2. 预测阶段（到 500Hz 节拍时）
   - 从 buffer 取 acc/gyro/thrust 平均值
   - 调用 kalmanCorePredict
3. 过程噪声注入
   - 按实际 dt 调用 kalmanCoreAddProcessNoise
4. 定时量测更新
   - Baro（若启用）
   - 加速度姿态校正（25Hz）
5. 队列量测更新（while pop）
   - TOF
   - YawError
   - AbsoluteHeight
   - Distance
   - Position
   - Pose
   - TDOA
   - Velocity
6. Finalize
   - 将 D0/D1/D2 融合入 q
   - 更新 R
   - 清零 D0/D1/D2
   - 监督器检查越界并触发重置
7. Externalize
   - 导出到 taskEstimatorState，供控制环读取

## 7. 预测模型（当前代码）

当前位置和速度传播使用 IMU 主导的运动学模型：

- 位置更新：
  x = x + R * v_body * dt + 0.5 * R * a_body * dt^2 - 0.5 * g * dt^2
- 速度更新：
  v_body = v_body + dt * (a_body + omega x v_body - R^T * g)
- 姿态更新：
  用角速度积分增量四元数，再乘到当前 q 上

重要说明：

- kalmanCorePredict 接口保留了 cmdThrust 参数
- 但当前预测方程本身未使用 cmdThrust 做动力学传播
- thrustAverage 目前主要用于 quadIsFlying 判定

## 8. 量测更新（当前有效路径）

### 8.1 TOF 高度更新

- 只在姿态几何可用时更新（R[2][2] 条件）
- 首次有效 TOF 会初始化高度状态
- 使用倾斜补偿的测量模型
- 对 innovation 做跳变保护：
  - > 0.3m：临时增大测量噪声
  - > 1.0m：重置高度状态

### 8.2 MTF01 速度更新

当前项目不是旧版 dpixel 光流更新链路，而是：

- 在 flow_mtf01 驱动中先做过滤与坐标映射
- 结合高度缩放得到机体系 velX/velY
- 通过 estimatorEnqueueVelocity 入队
- 在 kalmanCoreUpdateWithVelocity 中直接观测 PX/PY

### 8.3 加速度姿态更新（25Hz）

- 用重力方向约束 roll/pitch（不约束 yaw）
- 当 |acc|-1g 偏差变大时，自动增大测量噪声，降低更新强度

### 8.4 Baro 更新（代码存在，默认编译不开）

- 由 KALMAN_USE_BARO_UPDATE 控制
- 开启后先做 50 样本参考高度校准
- 用相对高度更新 Z

## 9. 项目特化改动（相对原始实现）

当前源码包含若干工程化增强：

- ROLLPITCH_ZERO_REVERSION 设置为 0（禁用自动回正）
- TOF 首次初始化高度 + 大跳变重置
- 陀螺仪 Z 轴 LMS 自适应串扰补偿（kxz/kyz/bz）
- MTF01 路径改为直接速度量测，不再走旧 dpixel 预测日志链
- 过程噪声与测量噪声参数按当前硬件进行了重新调整

## 10. 日志与参数（调试入口）

### 10.1 关键日志组

- kalman
  - 状态：stateX/stateY/stateZ/statePX/statePY/statePZ/stateD0/D1/D2
  - 协方差对角：varX/varY/varZ/varPX/varPY/varPZ/varD0/varD1/varD2
  - 四元数：q0/q1/q2/q3
  - 频率计数：rtPred/rtUpdate/rtBaro/rtFinal/rtApnd/rtRej
- kalman_baro
  - height（校准后 Baro 相对高度）

### 10.2 关键参数

- accAttStd0 / accAttStdK
  - 加速度姿态校正噪声模型参数
- pNAcc_xy / pNAcc_z / pNVel / pNPos / pNAtt
  - 过程噪声相关
- mNBaro / flowStdDev / tofStdDev
  - 量测噪声相关
- mNGyro_rollpitch / mNGyro_yaw
  - 姿态误差过程噪声项
- resetEstimation
  - 手动触发滤波器重置

## 12. 一句话总结

当前工程的卡尔曼估计器是“500Hz IMU 预测 + TOF/速度等异步量测更新 + 误差状态收敛导出”的分层实现；默认偏室内与近地工况，Baro 融合能力保留但默认不开启。

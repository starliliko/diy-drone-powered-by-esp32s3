# ESP-Drone 远程服务器通信配置指南

## 功能概述

本功能实现了 ESP-Drone 通过 Wi-Fi STA+AP 模式连接到路由器，并与远程服务器进行双向通信：

- **上行**：无人机定期上报遥测数据（姿态、位置、电池等）
- **下行**：服务器发送控制指令，无人机执行
- **兼容**：保留原有 AP 模式，手机可以同时直连控制

## 配置步骤

### 1. 运行 menuconfig

```powershell
idf.py menuconfig
```

### 2. 配置 Wi-Fi STA+AP 模式

进入 `diy-drone config` → `wireless config` → `STA Mode & Remote Server`：

| 配置项                       | 说明             | 示例值        |
| ---------------------------- | ---------------- | ------------- |
| `Enable STA+AP Mode`         | 启用 STA+AP 模式 | `[*]` (勾选)  |
| `Router SSID to connect`     | 路由器 SSID      | `MyHomeWiFi`  |
| `Router Password`            | 路由器密码       | `password123` |
| `STA connection retry count` | 连接重试次数     | `10`          |

### 3. 配置远程服务器

| 配置项                            | 说明             | 示例值          |
| --------------------------------- | ---------------- | --------------- |
| `Enable Remote Server Connection` | 启用远程服务器   | `[*]` (勾选)    |
| `Remote Server IP Address`        | 服务器 IP        | `192.168.1.100` |
| `Remote Server Port`              | 服务器端口       | `8080`          |
| `Reconnect interval`              | 断线重连间隔(秒) | `5`             |
| `Heartbeat interval`              | 心跳间隔(ms)     | `1000`          |
| `Telemetry upload interval`       | 遥测上报间隔(ms) | `100`           |

### 4. 编译烧录

```powershell
idf.py build
idf.py -p COM3 flash monitor
```

## 通信协议

### 数据包格式

```
┌───────┬───────┬─────────┬──────┬─────┬────────┬─────────────┐
│ Magic │ Magic │ Version │ Type │ Seq │ Length │   Payload   │
│ 0xAB  │ 0xCD  │  (1B)   │ (1B) │(2B) │  (2B)  │  (变长)     │
└───────┴───────┴─────────┴──────┴─────┴────────┴─────────────┘
```

### 包类型定义

| Type | 名称      | 方向 | 说明       |
| ---- | --------- | ---- | ---------- |
| 0x00 | HEARTBEAT | 双向 | 心跳保活   |
| 0x01 | TELEMETRY | 上行 | 遥测数据   |
| 0x02 | CONTROL   | 下行 | 控制指令   |
| 0x03 | CRTP      | 双向 | CRTP 透传  |
| 0x04 | ACK       | 双向 | 确认响应   |
| 0x05 | CONFIG    | 下行 | 运行时配置 |
| 0x06 | LOG       | 上行 | 日志消息   |

### 遥测数据结构 (TELEMETRY)

```c
typedef struct __attribute__((packed)) {
    int16_t roll;           // 姿态角 (度 * 100)
    int16_t pitch;
    int16_t yaw;
    int16_t gyroX;          // 角速度 (deg/s * 10)
    int16_t gyroY;
    int16_t gyroZ;
    int16_t accX;           // 加速度 (mg)
    int16_t accY;
    int16_t accZ;
    int32_t posX;           // 位置 (mm)
    int32_t posY;
    int32_t posZ;
    int16_t velX;           // 速度 (mm/s)
    int16_t velY;
    int16_t velZ;
    uint16_t battVoltage;   // 电池电压 (mV)
    uint8_t  battPercent;   // 电池百分比
    uint8_t  flightMode;    // 飞行模式
    uint8_t  isArmed;       // 是否解锁
    uint8_t  isLowBattery;  // 低电量警告
    uint32_t timestamp;     // 时间戳 (ms)
} RemoteTelemetryData;
```

### 控制指令结构 (CONTROL)

```c
typedef struct __attribute__((packed)) {
    uint8_t  cmdType;       // 指令类型 (见下表)
    int16_t  roll;          // 横滚角 (度*100)
    int16_t  pitch;         // 俯仰角 (度*100)
    int16_t  yaw;           // 偏航速率 (deg/s*10)
    uint16_t thrust;        // 油门 (0-65535)
    uint8_t  mode;          // 控制模式
    uint8_t  reserved[3];
} RemoteControlCmd;
```

| cmdType | 名称      | 说明                       |
| ------- | --------- | -------------------------- |
| 0x00    | RPYT      | Roll/Pitch/Yaw/Thrust 模式 |
| 0x01    | VELOCITY  | 速度控制模式               |
| 0x02    | POSITION  | 位置控制模式               |
| 0x03    | HOVER     | 悬停                       |
| 0x04    | LAND      | 降落                       |
| 0x05    | EMERGENCY | 紧急停机                   |
| 0x06    | ARM       | 解锁                       |
| 0x07    | DISARM    | 上锁                       |

## 扩展开发

### 自定义遥测数据

修改 `remote_server.c` 中的 `remoteServerTelemetryTask()`：

```c
// 从估计器获取姿态数据
telemetry.roll = (int16_t)(stateEstimatorGetRoll() * 100);
telemetry.pitch = (int16_t)(stateEstimatorGetPitch() * 100);
telemetry.yaw = (int16_t)(stateEstimatorGetYaw() * 100);

// 从电源管理获取电池数据
telemetry.battVoltage = pmGetBatteryVoltage() * 1000;
telemetry.battPercent = pmGetBatteryLevel();
```

### 注册控制回调

```c
void myControlHandler(const RemoteControlCmd* cmd) {
    // 自定义控制逻辑
}

// 在初始化时注册
remoteServerSetControlCallback(myControlHandler);
```

### 发送自定义数据

```c
// 发送日志
const char* msg = "Hello from drone!";
remoteServerSendPacket(REMOTE_PKT_LOG, (uint8_t*)msg, strlen(msg));

// 发送 CRTP 透传
CRTPPacket p;
// ... 构造包
remoteServerSendCRTP(p.raw, p.size + 1);
```

## 注意事项

1. **网络安全**：当前实现没有加密，仅适用于内网或测试环境。生产环境建议添加 TLS/DTLS。

2. **信道冲突**：STA 和 AP 必须使用相同的 Wi-Fi 信道。如果路由器使用固定信道，确保 Kconfig 中的 `WIFI_CHANNEL` 与之匹配。

3. **延迟考虑**：远程控制会有网络延迟，不适合实时飞行控制。建议仅用于：
   - 遥测监控
   - 航点任务下发
   - 状态查询
   - 紧急停机

4. **优先级**：本地 UDP 控制优先级与远程控制相同（COMMANDER_PRIORITY_CRTP）。可根据需要调整。

5. **断线处理**：服务器断线后会自动重连，重连间隔可配置。

## 故障排查

### 无法连接路由器

1. 检查 SSID 和密码是否正确
2. 确认路由器 2.4GHz 频段可用
3. 查看串口日志中的错误信息
4. 尝试增加重试次数

### 无法连接服务器

1. 确认服务器 IP 和端口正确
2. 检查防火墙设置
3. 确认 ESP32 已获取 IP 地址（日志中会打印）
4. 使用 `ping` 测试网络连通性

### 数据不更新

1. 检查遥测间隔配置
2. 确认服务器正确解析协议
3. 查看心跳是否正常

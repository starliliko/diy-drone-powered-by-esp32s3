# BMI088 加速度计数据不更新问题调试记录

## 问题描述

**现象**：BMI088 加速度计初始化后，数据读取只有一次有效值，之后数据不再更新（始终返回相同的值）。陀螺仪数据正常更新。

**日志表现**：
```
ACC data test: [0] X=-5362 Y=-1046 Z=-690
ACC data test: [1] X=-5362 Y=-1046 Z=-690  ← 数据不变
ACC data test: [2] X=-5362 Y=-1046 Z=-690  ← 数据不变
E (841) BMI088_SPI: ACC DATA NOT UPDATING! Accelerometer may be in SUSPEND mode!
```

**关键诊断信息**：
- `INT_STAT_1 = 0x00`（数据就绪标志始终为0）
- `ACC_ERR_REG = 0x04`（error_code=1，表示配置错误）
- 陀螺仪 `INT_STAT_1 = 0x80`（正常）

---

## 调试过程

### 第一阶段：中断配置排查

**检查点**：
1. INT1 引脚配置是否正确
2. 中断极性和触发模式

**发现的问题**：
- 原代码配置 INT1 为 active-HIGH，但 BMI088 默认是 active-LOW
- ESP32 GPIO 配置为 `GPIO_INTR_HIGH_LEVEL`，与传感器极性不匹配

**修复**：
```c
// 修改前
uint8_t int1_io_conf = 0x0A;  // active HIGH

// 修改后
uint8_t int1_io_conf = 0x08;  // active LOW, push-pull, output enabled
```

```c
// ESP32 GPIO 配置修改
gpio_config.intr_type = GPIO_INTR_LOW_LEVEL;  // 改为低电平触发
gpio_config.pull_up_en = GPIO_PULLUP_ENABLE;   // 添加上拉
```

**结果**：中断极性问题已修复，但数据仍不更新。

---

### 第二阶段：电源时序排查

**检查点**：
1. PWR_CONF 和 PWR_CTRL 寄存器配置顺序
2. 上电等待时间是否足够

**发现的问题**：
- 软复位后没有重新激活 SPI 模式（BMI088 复位后默认回到 I2C 模式）
- PWR_CTRL 写入后等待时间不足（需要 50ms）

**修复**：
```c
// 软复位后重新激活 SPI 模式
bmi088_acc_read_reg(dev, BMI088_ACC_CHIP_ID_REG, &dummy, 1);  // 第一次读取切换到SPI
BMI088_DELAY_MS(2);
bmi088_acc_read_reg(dev, BMI088_ACC_CHIP_ID_REG, &dummy, 1);  // 第二次确认
BMI088_DELAY_MS(2);

// 正确的电源配置顺序
bmi088_acc_write_reg(dev, BMI088_ACC_PWR_CONF_REG, 0x00, 1);  // 1. 退出suspend
BMI088_DELAY_MS(1);  // 等待450us以上
bmi088_acc_write_reg(dev, BMI088_ACC_PWR_CTRL_REG, 0x04, 1);  // 2. 使能加速度计
BMI088_DELAY_MS(50);  // 【关键】等待50ms让模拟电路稳定
```

**结果**：寄存器读回值正确，但数据仍不更新。

---

### 第三阶段：ACC_CONF 寄存器配置排查（根本原因）

**关键诊断信息**：
```
ACC_ERR_REG(0x02)=0x04, ACC_STATUS(0x03)=0x90
ACC has error! error_code=1, fatal=0
```

`error_code=1` 表示**配置错误**！

**检查 ACC_CONF 寄存器格式**：

根据 BMI088 数据手册，ACC_CONF (0x40) 寄存器结构：
| Bit | 名称         | 说明                        |
| --- | ------------ | --------------------------- |
| 7   | acc_bwp[3]   | 带宽参数高位，**必须设为1** |
| 6:4 | acc_bwp[2:0] | 带宽参数低3位               |
| 3:0 | acc_odr      | 输出数据率                  |

**发现的问题**：
原代码计算的 ACC_CONF 值：
```c
uint8_t conf_val = ((uint8_t)odr << 0) | ((uint8_t)bwp << 4);
// odr = 0x0C (1600Hz), bwp = 0x02 (Normal)
// conf_val = 0x2C = 0b00101100  ← bit7 = 0，错误！
```

Crazyflie 参考代码使用的值：**0xAC** = `0b10101100`（bit7 = 1）

**根本原因**：ACC_CONF 寄存器的 **bit7 必须设置为 1**，否则加速度计不会正常采样！

**修复**：
```c
// 修改前
uint8_t conf_val = ((uint8_t)odr << 0) | ((uint8_t)bwp << 4);  // = 0x2C

// 修改后
uint8_t conf_val = 0x80 | ((uint8_t)bwp << 4) | (uint8_t)odr;  // = 0xAC
```

---

## 最终解决方案

### 关键代码修改

**文件**：`components/drivers/spi_devices/bmi088/bmi088_spi.c`

#### 1. ACC_CONF 寄存器配置（核心修复）
```c
// Step 3: 配置ACC_CONF（ODR + 带宽）
// 【关键】bit7 必须设为 1，否则加速度计不工作
uint8_t conf_val = 0x80 | ((uint8_t)bwp << 4) | (uint8_t)odr;
ESP_LOGI(TAG, "Writing ACC_CONF: 0x%02X (odr=%d, bwp=%d)", conf_val, odr, bwp);
bmi088_acc_write_reg(dev, BMI088_ACC_CONF_REG, &conf_val, 1);
BMI088_DELAY_MS(10);
```

#### 2. 软复位后 SPI 模式激活
```c
// 软复位后必须重新激活 SPI 模式
uint8_t dummy;
bmi088_acc_read_reg(dev, BMI088_ACC_CHIP_ID_REG, &dummy, 1);  // 切换到SPI
BMI088_DELAY_MS(2);
bmi088_acc_read_reg(dev, BMI088_ACC_CHIP_ID_REG, &dummy, 1);  // 确认
BMI088_DELAY_MS(2);
bmi088_acc_read_reg(dev, BMI088_ACC_CHIP_ID_REG, &dummy, 1);  // 验证
```

#### 3. 正确的电源配置时序
```c
// PWR_CONF = 0x00 (退出suspend)
bmi088_acc_write_reg(dev, BMI088_ACC_PWR_CONF_REG, &pwr_conf, 1);
BMI088_DELAY_MS(1);  // 等待450us以上

// PWR_CTRL = 0x04 (使能加速度计)
bmi088_acc_write_reg(dev, BMI088_ACC_PWR_CTRL_REG, &pwr_ctrl, 1);
BMI088_DELAY_MS(50);  // 【关键】等待50ms
```

#### 4. 中断配置（INT1 active-LOW）
```c
// INT1: active LOW, push-pull, output enabled
uint8_t int1_io_conf = 0x08;  // bit3=1(output), bit2=0(push-pull), bit1=0(active-low)
bmi088_acc_write_reg(dev, BMI088_ACC_INT1_IO_CONF_REG, &int1_io_conf, 1);
```

## 经验总结

### BMI088 加速度计初始化要点

1. **SPI 模式激活**：上电/复位后必须先执行一次 SPI 读取来切换模式
2. **电源时序**：
   - 先写 PWR_CONF=0x00（退出suspend）
   - 等待 450us 以上
   - 再写 PWR_CTRL=0x04（使能加速度计）
   - **等待 50ms**（让模拟电路稳定）
3. **ACC_CONF bit7**：必须设为 1，否则加速度计不会正常采样
4. **中断极性**：BMI088 默认 active-LOW，ESP32 GPIO 需配置匹配
5. **不要使用 0x59 配置文件加载**：会导致初始化失败

### 关键寄存器值

| 寄存器        | 地址 | 正确值 | 说明                              |
| ------------- | ---- | ------ | --------------------------------- |
| ACC_CONF      | 0x40 | 0xAC   | 1600Hz ODR, Normal BW, **bit7=1** |
| ACC_RANGE     | 0x41 | 0x01   | 6G 量程                           |
| PWR_CONF      | 0x7C | 0x00   | Active 模式                       |
| PWR_CTRL      | 0x7D | 0x04   | 加速度计使能                      |
| INT1_IO_CONF  | 0x53 | 0x08   | active-LOW, push-pull             |
| INT1_INT2_MAP | 0x58 | 0x04   | DRDY 映射到 INT1                  |

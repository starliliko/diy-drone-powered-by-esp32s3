# BMI088 传感器坐标轴映射修正

## 问题描述

在 ESP-Drone 飞控板上，使用 BMI088 SPI 传感器时，姿态数据输出存在坐标轴映射错误：

| 问题                | 实测行为                                                                        | 期望行为               |
| ------------------- | ------------------------------------------------------------------------------- | ---------------------- |
| **Roll/Pitch 交换** | `state.attitude.roll` 输出 pitch 数据<br/>`state.attitude.pitch` 输出 roll 数据 | Roll 和 Pitch 正确对应 |
| **Pitch 方向反转**  | 抬头为负值，低头为正值                                                          | 抬头为正值，低头为负值 |
| **Roll 方向**       | 右倾为正 ✓（正确）                                                              | 右倾为正               |
| **Yaw 方向反转**    | 顺时针为负值                                                                    | 顺时针为正值           |

### 测试输出示例

**修改前**（错误）：
```c
printf("%.2f,%.2f,%.2f\n", state.attitude.roll, state.attitude.pitch, state.attitude.yaw);
// 输出: roll值显示pitch, pitch值显示roll, pitch抬头为负, yaw顺时针为负
```

---

## 根本原因

BMI088 传感器的物理安装方向与 Crazyflie 标准机体坐标系存在轴向差异：

1. **X/Y 轴映射错误**：传感器的 X 轴对应飞控的 Y 轴（Pitch），Y 轴对应 X 轴（Roll）
2. **Pitch 轴反向**：传感器 Pitch 轴（原 X 轴）方向与飞控期望相反
3. **Yaw 轴反向**：传感器 Z 轴旋转方向与飞控右手定则相反

---

## 解决方案

### 修改文件

**文件路径**: `components/core/crazyflie/hal/src/sensors_bmi088_spi_ms5611.c`

**修改位置**: 约第 262-275 行（陀螺仪和加速度计数据处理部分）

### 修改前代码

```c
// 陀螺仪数据处理
sensorData.gyro.x = (gyroRaw.x - gyroBias.x) * SENSORS_BMI088_DEG_PER_LSB_CFG;
sensorData.gyro.y = (gyroRaw.y - gyroBias.y) * SENSORS_BMI088_DEG_PER_LSB_CFG;
sensorData.gyro.z = (gyroRaw.z - gyroBias.z) * SENSORS_BMI088_DEG_PER_LSB_CFG;
applyAxis3fLpf((lpf2pData *)(&gyroLpf), &sensorData.gyro);

// 加速度计数据处理
accScaled.x = accelRaw.x * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
accScaled.y = accelRaw.y * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
accScaled.z = accelRaw.z * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
```

### 修改后代码

```c
// 陀螺仪数据处理
// 坐标轴映射修正：
//   - 原roll输出pitch，pitch输出roll → 交换x和y
//   - pitch抬头应为正 → 取反y(交换后)
//   - yaw顺时针应为正 → 取反z
sensorData.gyro.x = -(gyroRaw.y - gyroBias.y) * SENSORS_BMI088_DEG_PER_LSB_CFG;  // pitch轴(原y) → roll，取反
sensorData.gyro.y = (gyroRaw.x - gyroBias.x) * SENSORS_BMI088_DEG_PER_LSB_CFG;   // roll轴(原x) → pitch
sensorData.gyro.z = -(gyroRaw.z - gyroBias.z) * SENSORS_BMI088_DEG_PER_LSB_CFG;  // yaw轴取反
applyAxis3fLpf((lpf2pData *)(&gyroLpf), &sensorData.gyro);

// 加速度计数据处理 - 轴映射与陀螺仪一致
accScaled.x = -accelRaw.y * SENSORS_BMI088_G_PER_LSB_CFG / accScale;  // 交换并取反
accScaled.y = accelRaw.x * SENSORS_BMI088_G_PER_LSB_CFG / accScale;   // 交换
accScaled.z = accelRaw.z * SENSORS_BMI088_G_PER_LSB_CFG / accScale;
```

## 轴映射对照表

### 陀螺仪轴映射

| 飞控坐标轴       | 修改前      | 修改后       | 说明                     |
| ---------------- | ----------- | ------------ | ------------------------ |
| `gyro.x` (Roll)  | `gyroRaw.x` | `-gyroRaw.y` | 交换 X↔Y，取反使右倾为正 |
| `gyro.y` (Pitch) | `gyroRaw.y` | `gyroRaw.x`  | 交换 X↔Y，使抬头为正     |
| `gyro.z` (Yaw)   | `gyroRaw.z` | `-gyroRaw.z` | 取反使顺时针为正         |

### 加速度计轴映射

| 飞控坐标轴 | 修改前       | 修改后        | 说明             |
| ---------- | ------------ | ------------- | ---------------- |
| `acc.x`    | `accelRaw.x` | `-accelRaw.y` | 与陀螺仪保持一致 |
| `acc.y`    | `accelRaw.y` | `accelRaw.x`  | 与陀螺仪保持一致 |
| `acc.z`    | `accelRaw.z` | `accelRaw.z`  | 保持不变         |

## 技术细节

### 坐标系定义

**Crazyflie/ESP-Drone 机体坐标系** (FRD - Forward-Right-Down)：

```
      机头 (前)
         ↑
         X
         |
    Y ←--⊗ (机体中心)
        /
       Z (下)

Roll:  绕X轴旋转，右倾为正
Pitch: 绕Y轴旋转，抬头为正
Yaw:   绕Z轴旋转，顺时针为正（俯视）
```

### 为什么需要同时修改陀螺仪和加速度计

1. **姿态融合算法要求**：互补滤波/卡尔曼滤波需要陀螺仪和加速度计的坐标系一致
2. **传感器校准**：`sensorsAccAlignToGravity()` 使用加速度计数据进行重力对齐
3. **状态估计精度**：坐标系不一致会导致姿态估计发散

## 注意事项

1. **校准重置**：修改轴映射后，建议重新执行陀螺仪校准（飞控会自动进行）
2. **PID 参数**：如果之前针对错误坐标系调整过 PID 参数，需要重新调整
3. **飞行测试**：首次飞行前务必在测试架上验证控制方向正确性
4. **硬件差异**：本修改仅适用于 BMI088 SPI 传感器，MPU6050/MPU9250 等传感器可能需要不同的映射

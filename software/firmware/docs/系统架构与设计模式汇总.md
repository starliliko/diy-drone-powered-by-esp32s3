## 系统架构概览

### 系统初始化流程

```
main.c
  └─> platformInit()
      └─> systemLaunch()
          └─> systemTask (FreeRTOS 任务)
              ├─> 底层外设初始化 (LED, WiFi)
              ├─> 高级模块初始化 (System, Comm, Commander, SBUS, Stabilizer)
              ├─> 模块自检 (各模块 Test 函数)
              ├─> systemStart() (释放启动信号量)
              └─> workerLoop() (异步任务处理循环)
```

### 核心组件结构

```
components/
├── core/crazyflie/           # 飞控核心逻辑
│   ├── hal/                  # 硬件抽象层 (传感器、LED、WiFi)
│   ├── modules/              # 飞行模块
│   │   ├── stabilizer.c      # 稳定器 (1000Hz 控制循环)
│   │   ├── commander.c       # 指令管理器 (setpoint 分发)
│   │   ├── estimator_kalman.c # 卡尔曼滤波状态估计
│   │   ├── controller_pid.c  # PID 控制器
│   │   ├── extrx.c          # 外部接收器 (SBUS)
│   │   ├── worker.c         # 异步任务调度
│   │   └── system.c         # 系统主任务
│   └── utils/               # 工具函数
├── drivers/                 # 设备驱动
│   ├── i2c_devices/        # MS5611 气压计、QMC5883P 磁力计、EEPROM
│   ├── spi_devices/        # BMI088 IMU
│   └── general/            # 电机 PWM、WiFi、SBUS、MTF01 光流
└── platform/               # 硬件平台抽象
```

## 核心设计模式

### 1. 生产者-消费者模式 (Producer-Consumer Pattern)

**应用场景**: `worker.c` 异步任务调度系统

```
生产者 (任意任务)                 消费者 (workerLoop)
      │                                 │
      ├─ stabilizerTask               │
      ├─ commTask            队列      │
      ├─ extRxTask      ┌──────────┐  │
      │                 │          │  │
      └──workerSchedule→│ Queue(5) │──┤
                        │          │  │
                        └──────────┘  ▼
                                   workerLoop()
                                   执行任务
```

**关键代码**:
```c
// 生产者接口
int workerSchedule(void (*function)(void*), void *arg);

// 消费者循环
void workerLoop() {
  while (1) {
    xQueueReceive(workerQueue, &work, portMAX_DELAY);
    work.function(work.arg);
  }
}
```

**知识点**: 操作系统 - 进程同步、队列通信

---

### 2. 命令模式 (Command Pattern)

**应用场景**: 将函数调用封装为可传递的对象

```c
struct worker_work {
  void (*function)(void*);  // 命令（操作）
  void* arg;                // 参数（上下文）
};
```

**优势**:
- 操作可以被存储、排队、延迟执行
- 调用者和执行者解耦

**知识点**: 设计模式、函数指针、回调机制

---

### 3. 优先级调度与实时性保障

**应用场景**: FreeRTOS 任务优先级设计

```
优先级 7 (最高) ──> stabilizerTask   (1 kHz 飞控主循环)
优先级 6        ──> sensorsTask      (1 kHz 传感器读取)
优先级 4        ──> estimatorKalman  (500 Hz EKF 预测)
优先级 3        ──> commander / extRx (setpoint / SBUS)
优先级 2        ──> remoteServerRx/Tx (MAVLink/CRTP over WiFi)
优先级 1 (最低) ──> systemTask / workerLoop (异步任务)
```

**设计原则**:
- 高优先级任务不执行耗时操作
- 耗时操作通过 `workerSchedule()` 委托给低优先级任务
- 避免优先级反转

**知识点**: 操作系统 - 任务调度、实时系统设计

---

### 4. 分层架构

```
应用层         commander.c         (接收 setpoint)
                    ↓
控制层         controller_pid.c    (PID 控制算法)
                    ↓
估计层         estimator_kalman.c  (状态估计)
                    ↓
驱动层         sensors.c, motors.c (硬件接口)
                    ↓
硬件层         BMI088 (SPI), MS5611 + QMC5883P (I2C), 电机 PWM
```

**知识点**: 软件工程 - 模块化设计、分层架构

## FreeRTOS 多任务架构

### ESP32 双核 SMP 模式

```
ESP32-S3 双核架构
├── CPU0 (PRO_CPU)
│   ├── stabilizerTask (自动调度)
│   ├── extRxTask
│   └── ...
└── CPU1 (APP_CPU)
    ├── sensorsTask (自动调度)
    ├── remoteServerTx / remoteServerRx
    └── ...
```

**特性**:
- SMP (对称多处理): 两个核心地位相等
- FreeRTOS 自动将任务分配到两个核心
- 使用 `xTaskCreate` (非固定核心) 或 `xTaskCreatePinnedToCore` (固定核心)

**知识点**: 操作系统 - 多核架构、SMP vs AMP

---

### 任务同步机制

| 机制                   | 用途               | ESP-Drone 中的应用             |
| ---------------------- | ------------------ | ------------------------------ |
| **信号量 (Semaphore)** | 资源计数、事件通知 | `canStartMutex` (启动同步)     |
| **互斥锁 (Mutex)**     | 保护共享资源       | I2C 总线互斥访问               |
| **队列 (Queue)**       | 任务间通信         | `workerQueue`, `setpointQueue` |
| **中断控制**           | 临界区保护         | `portENTER_CRITICAL()`         |

**知识点**: 操作系统 - 进程同步、临界区

---

### systemStart() 启动同步机制

```c
// 初始化时创建互斥信号量并获取
xSemaphoreTake(canStartMutex, portMAX_DELAY);

// 其他任务等待启动
void systemWaitStart(void) {
  xSemaphoreTake(canStartMutex, portMAX_DELAY);
  xSemaphoreGive(canStartMutex);  // 立即释放
}

// 初始化完成后释放信号量
void systemStart() {
  xSemaphoreGive(canStartMutex);
}
```

**效果**: 所有任务阻塞等待，直到系统初始化完成

## 控制流与数据流

### 飞控主循环 (1000Hz)

```
sensorsTask (1 kHz)
    ↓ 读取传感器数据
sensors.c → BMI088 (IMU, SPI), MS5611 (气压计), QMC5883P (磁力计)
    ↓ 原始数据
estimatorKalmanTask
    ↓ 卡尔曼滤波融合
state_t (姿态、位置、速度估计)
    ↓
stabilizerTask (1000Hz)
    ↓ 获取 setpoint
commanderGetSetpoint()
    ↓
controller_pid.c (PID 控制)
    ↓
motorsSetRatio() (PWM 输出)
    ↓
电机旋转 → 飞行
```

---

### 控制指令优先级机制

```c
// commander.h
#define COMMANDER_PRIORITY_DISABLE 0  // 禁用
#define COMMANDER_PRIORITY_CRTP    1  // WiFi/手机APP
#define COMMANDER_PRIORITY_EXTRX   2  // 遥控器 (最高)
```

**数据流**:

```
手机APP (WiFi)                    遥控器 (SBUS)
    ↓                                ↓
crtpCommanderCB()                extRxTask()
    ↓                                ↓
commanderSetSetpoint(优先级1)    commanderSetSetpoint(优先级2)
    ↓                                ↓
          ┌──────────────────────────┐
          │  setpointQueue (队列)     │
          │  只有优先级 >= 当前优先级  │
          │  才能覆盖                 │
          └──────────────────────────┘
                    ↓
          commanderGetSetpoint()
                    ↓
              stabilizerTask
```

**安全设计**: 遥控器始终可以覆盖手机APP指令

---

### SBUS 接收器数据流

```
FS-I6X 遥控器
    ↓ 2.4GHz 无线
IA-10B 接收器
    ↓ SBUS 协议 (100000 baud, 8E2, 串口)
sbusReadFrame()  (sbus.c)
    ↓ 解析 16 通道数据
extRxDecodeChannels()  (extrx.c)
    ↓ 转换为 setpoint
    │ • Roll:  -20° ~ +20°
    │ • Pitch: -20° ~ +20°
    │ • Yaw:   -150°/s ~ +150°/s
    │ • Thrust: 0 ~ 65535
    ↓
commanderSetSetpoint(&extrxSetpoint, COMMANDER_PRIORITY_EXTRX)
```

## 关键知识点

### 1. 多态 (Polymorphism)

**定义**: 同一接口，不同实现，运行时决定调用哪个

**C语言实现** (函数指针):
```c
void (*function)(void*);  // 函数指针可以指向不同函数
work.function = saveConfig;
work.function = sendData;
work.function(arg);  // 运行时决定调用哪个
```

**C++实现** (虚函数):
```cpp
class Animal { virtual void speak() = 0; };
class Dog : public Animal { void speak() { cout << "汪"; } };
class Cat : public Animal { void speak() { cout << "喵"; } };
```

**知识来源**: 面向对象程序设计 (OOP)

---

### 2. 线程池 (Thread Pool)

**定义**: 预创建一组线程，重复利用执行任务

**标准线程池**:
```
任务队列 → [ Worker1, Worker2, Worker3, ... ] → 并行执行
```

**ESP-Drone Worker** (简化版):
```
任务队列 → [ Worker (单线程) ] → 串行执行
```

**为什么简化?**
- 嵌入式内存有限
- 串行执行足够应对非实时任务
- 降低复杂度，提高可靠性

**知识来源**: 操作系统、并发编程

---

### 3. CRTP 协议 (Crazy RealTime Protocol)

**定义**: Crazyflie 自定义的轻量级通信协议

**数据包结构**:
```c
typedef struct {
  uint8_t header;     // 端口 + 通道
  uint8_t size;       // 数据长度
  uint8_t data[30];   // 载荷
} CRTPPacket;
```

**端口分配**:
- `CRTP_PORT_CONSOLE` (0) - 调试输出
- `CRTP_PORT_PARAM` (2) - 参数读写
- `CRTP_PORT_SETPOINT` (3) - 控制指令
- `CRTP_PORT_MEM` (4) - 内存管理
- `CRTP_PORT_LOG` (5) - 日志数据

**传输层**: WiFi UDP / TCP,统一由 `remote_server.c` 实现(MAVLink v2 + 私有 0xABCD 双栈)

---

### 4. 参数与日志系统

**参数系统** (可读写):
```c
PARAM_GROUP_START(controller)
PARAM_ADD(PARAM_FLOAT, kp_roll, &kp_roll)
PARAM_ADD(PARAM_FLOAT, ki_roll, &ki_roll)
PARAM_GROUP_STOP(controller)
```

**日志系统** (只读):
```c
LOG_GROUP_START(stabilizer)
LOG_ADD(LOG_FLOAT, roll, &state.attitude.roll)
LOG_ADD(LOG_FLOAT, pitch, &state.attitude.pitch)
LOG_GROUP_STOP(stabilizer)
```

**应用**: 通过 cfclient (PC客户端) 实时调参和监控

---

### 5. 静态内存分配

**FreeRTOS 两种创建方式**:

| 方式     | 函数                | 内存分配   |
| -------- | ------------------- | ---------- |
| 动态分配 | `xTaskCreate`       | 从堆分配   |
| 静态分配 | `xTaskCreateStatic` | 编译时分配 |

**ESP-Drone 全部使用静态分配**:
```c
STATIC_MEM_TASK_ALLOC(systemTask, SYSTEM_TASK_STACKSIZE);
STATIC_MEM_TASK_CREATE(systemTask, systemTask, SYSTEM_TASK_NAME, NULL, SYSTEM_TASK_PRI);
```

**优势**:
- 避免堆碎片
- 可预测内存使用
- 提高可靠性

## 总结

ESP-Drone 是一个融合多种设计模式和系统架构思想的优秀嵌入式项目:

1. **实时性**: 通过 FreeRTOS 优先级调度保证 1000Hz 控制循环
2. **可靠性**: 静态内存分配、优先级设计、安全机制
3. **可扩展性**: 模块化设计、分层架构、驱动抽象
4. **高效性**: 双核 SMP、异步任务、队列通信

通过学习这个项目，可以深入理解操作系统、嵌入式系统、控制理论的实际应用。

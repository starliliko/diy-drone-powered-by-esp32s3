# ESP-Drone 配置系统说明

## 1. 配置文件体系

ESP-IDF 使用 Kconfig 配置系统（源自 Linux 内核），通过图形化菜单管理项目配置。

### 1.1 文件关系图

```
┌─────────────────────────────────────────────────────────────────┐
│                        配置定义层                                │
│                                                                  │
│   Kconfig.projbuild          Kconfig (组件)                      │
│   (项目配置菜单)              (组件配置菜单)                       │
│        │                          │                              │
│        └──────────┬───────────────┘                              │
│                   ↓                                              │
│            idf.py menuconfig                                     │
│            (图形化配置界面)                                       │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│                        配置存储层                                │
│                                                                  │
│   sdkconfig.defaults         sdkconfig.defaults.esp32s3         │
│   (通用默认值)                (芯片专用默认值)                    │
│        │                          │                              │
│        └──────────┬───────────────┘                              │
│                   ↓                                              │
│              sdkconfig                                           │
│        (当前实际配置，自动生成)                                   │
└─────────────────────────────────────────────────────────────────┘
                            │
                            ↓
┌─────────────────────────────────────────────────────────────────┐
│                        代码使用层                                │
│                                                                  │
│           build/config/sdkconfig.h                               │
│           (#define CONFIG_XXX 宏)                                │
│                   ↓                                              │
│            你的 C 代码中引用                                      │
│           #include "sdkconfig.h"                                 │
│           if (CONFIG_ENABLE_SBUS) { ... }                        │
└─────────────────────────────────────────────────────────────────┘
```

---

## 2. 各配置文件详解

| 文件                         | 位置          | 作用                                       | 谁创建     | 是否提交Git |
| ---------------------------- | ------------- | ------------------------------------------ | ---------- | ----------- |
| `Kconfig.projbuild`          | main/         | 定义项目级配置菜单，显示在 menuconfig 顶层 | 开发者手写 | ✅ 是        |
| `Kconfig`                    | 组件目录      | 定义组件级配置，显示在 Component config 下 | 开发者手写 | ✅ 是        |
| `sdkconfig.defaults`         | 项目根目录    | 所有芯片通用的默认配置                     | 开发者手写 | ✅ 是        |
| `sdkconfig.defaults.esp32s3` | 项目根目录    | ESP32-S3 专用默认配置                      | 开发者手写 | ✅ 是        |
| `sdkconfig`                  | 项目根目录    | 当前实际配置（menuconfig 保存结果）        | 自动生成   | ⚠️ 可选      |
| `sdkconfig.h`                | build/config/ | 将配置转为 C 宏供代码使用                  | 自动生成   | ❌ 否        |

## 3. Kconfig.projbuild vs Kconfig

| 特性     | Kconfig.projbuild            | Kconfig                 |
| -------- | ---------------------------- | ----------------------- |
| 显示位置 | menuconfig **顶层菜单**      | Component config 子菜单 |
| 加载条件 | **始终加载**                 | 组件被依赖时才加载      |
| 适用场景 | 项目全局配置（引脚、WiFi等） | 组件内部私有配置        |
| 可见性   | 用户第一眼就能看到           | 需要进入子菜单          |

### 本项目的 Kconfig.projbuild 结构

```
menu "ESPDrone Config"
├── choice "hardware version"     # 硬件版本选择
├── menu "app set"                # 应用设置
├── menu "wireless config"        # WiFi 配置
├── menu "calibration angle"      # 校准角度
├── menu "system"                 # 系统配置
├── menu "buzzer"                 # 蜂鸣器配置
├── menu "sensors config"         # 传感器引脚
├── menu "led config"             # LED 引脚
├── menu "motors config"          # 电机配置
└── menu "SBUS config"            # SBUS 遥控器配置
endmenu
```

## 4. 配置优先级

从高到低：

1. `sdkconfig` — 用户当前配置（最高优先级）
2. `sdkconfig.defaults.esp32s3` — 芯片专用默认值
3. `sdkconfig.defaults` — 通用默认值
4. Kconfig 中的 `default` — 代码定义的默认值

## 5. 使用流程

### 5.1 修改配置

```bash
# 方法1：图形化配置（推荐）
idf.py menuconfig

# 方法2：直接编辑 sdkconfig（不推荐）
# 方法3：修改 sdkconfig.defaults* 后重新构建
```

### 5.2 配置生效流程

```
1. 运行 idf.py menuconfig
2. 在图形界面中修改选项
3. 保存退出 → 更新 sdkconfig
4. 运行 idf.py build
5. 构建系统生成 sdkconfig.h
6. 代码中通过 CONFIG_XXX 宏使用配置
```

### 5.3 代码中使用配置

```c
#include "sdkconfig.h"

// 条件编译
#ifdef CONFIG_ENABLE_SBUS
    sbusInit();
#endif

// 直接使用宏值
gpio_num_t rx_pin = CONFIG_SBUS_RX_PIN;
```

## 6. 与 STM32CubeMX 对比

| 特性              | ESP-IDF Kconfig    | STM32CubeMX           |
| ----------------- | ------------------ | --------------------- |
| **生成内容**      | 仅 #define 宏      | 完整的 C 代码         |
| **代码编写**      | 完全手写，引用宏   | 自动生成 + 用户代码区 |
| **配置文件**      | sdkconfig (键值对) | .ioc (XML)            |
| **修改后**        | 增量编译           | 重新生成代码          |
| **灵活性**        | 完全自由           | 受限于生成模板        |
| **对 Linux 帮助** | ✅ 直接迁移         | ❌ 完全不同体系        |

---

## 7. 最佳实践

1. **项目配置**放在 `main/Kconfig.projbuild`
2. **组件私有配置**放在组件目录的 `Kconfig`
3. **默认值**写在 `sdkconfig.defaults*`，便于团队共享
4. **不要手动编辑** `sdkconfig`，用 menuconfig
5. **版本管理**时提交 `sdkconfig.defaults*`，`sdkconfig` 可选

## 8. 常用 Kconfig 语法

```kconfig
# 布尔选项
config ENABLE_FEATURE
    bool "Enable this feature"
    default n
    help
        Help text here.

# 整数选项
config PIN_NUMBER
    int "GPIO pin number"
    range 0 48
    default 18

# 字符串选项
config WIFI_SSID
    string "WiFi SSID"
    default "ESP-DRONE"

# 单选（choice）
choice
    prompt "Select option"
    default OPTION_A
    config OPTION_A
        bool "Option A"
    config OPTION_B
        bool "Option B"
endchoice

# 依赖关系
config SUB_OPTION
    bool "Sub option"
    depends on ENABLE_FEATURE
    default y if ENABLE_FEATURE

# 菜单分组
menu "Group Name"
    # 配置项...
endmenu
```

## 9. 总结

| 文件                         | 一句话总结                |
| ---------------------------- | ------------------------- |
| `Kconfig.projbuild`          | **定义菜单长什么样**      |
| `Kconfig`                    | **组件自己的配置菜单**    |
| `sdkconfig.defaults`         | **所有芯片的默认值**      |
| `sdkconfig.defaults.esp32s3` | **S3 芯片的专用默认值**   |
| `sdkconfig`                  | **用户实际选择的配置**    |
| `sdkconfig.h`                | **代码中用的 #define 宏** |

**记忆口诀**：  
Kconfig **定义**选项 → defaults **提供**默认值 → sdkconfig **保存**用户选择 → sdkconfig.h **生成**代码宏

# M2电机输出为0问题解决方案

## 问题概述

**现象：** ESP-Drone四轴飞行器M2电机输出始终为0，而M1/M3/M4电机输出异常，导致无法正常飞行。

**硬件配置：**
- 平台：ESP32-S3
- 传感器：BMI088 (SPI接口)
- 控制器：SBUS外部接收机
- 飞控框架：Crazyflie (ESP-IDF v5.5.1)
- 电机布局：X模式 (QUAD_FORMATION_X)

---

## 诊断过程

### 第一步：添加调试输出

在 `power_distribution_stock.c` 中添加详细的调试输出，显示：
- 四个电机的最终输出值 (M1-M4)
- 控制输入值 (Thrust, Roll, Pitch, Yaw)
- 限制前的原始计算值 (RAW)

```c
printf("[PWR] M1:%d M2:%d M3:%d M4:%d | T:%.0f R:%d P:%d Y:%d\n",
       motorPower.m1, motorPower.m2, motorPower.m3, motorPower.m4,
       control->thrust, control->roll, control->pitch, control->yaw);
printf("[RAW] M1:%d M2:%d M3:%d M4:%d\n",
       raw_m1, raw_m2, raw_m3, raw_m4);
```

**发现问题1：** Roll控制值高达2091-30000，远超正常范围(±500)

### 第二步：追踪PID控制器

在 `attitude_pid_controller.c` 中添加PID诊断输出：

```c
printf("[PID] RateActual: R:%.1f P:%.1f Y:%.1f | RateDesired: R:%.1f P:%.1f Y:%.1f | Output: R:%d P:%d Y:%d\n",
       rollRateActual, pitchRateActual, yawRateActual,
       rollRateDesired, pitchRateDesired, yawRateDesired,
       rollOutput, pitchOutput, yawOutput);
```

**发现问题2：**
- 陀螺仪读数正常 (RateActual: ±5°/s)
- 但PID输出持续饱和并发散 (Output: ±2000 → ±30000)

### 第三步：分析根本原因

通过观察PID输出与姿态响应的关系，发现：

1. **PID增益过大**
   - 原始参数：KP=250, KI=500, KD=2.5
   - 小的角速度误差就导致巨大输出

2. **控制方向形成正反馈**
   - Roll输出负值应该纠正姿态
   - 但实际姿态继续偏离 → 输出继续增大 → 发散
   - 说明控制方向与传感器框架不匹配

---

## 解决方案

### 修改1：大幅降低Rate PID参数

**文件：** `components/core/crazyflie/modules/interface/pid.h`

**修改内容：**
```c
// 原始参数（导致饱和）
#define PID_ROLL_RATE_KP  250.0
#define PID_ROLL_RATE_KI  500.0
#define PID_ROLL_RATE_KD  2.5

#define PID_PITCH_RATE_KP  250.0
#define PID_PITCH_RATE_KI  500.0
#define PID_PITCH_RATE_KD  2.5

// 修改后参数（稳定收敛）
#define PID_ROLL_RATE_KP  15.0   // 减小至1/16
#define PID_ROLL_RATE_KI  0.0    // 禁用积分防止windup
#define PID_ROLL_RATE_KD  3.0    // 增加阻尼

#define PID_PITCH_RATE_KP  15.0
#define PID_PITCH_RATE_KI  0.0
#define PID_PITCH_RATE_KD  3.0
```

**原理：** 降低比例增益可减少输出幅度，禁用积分项防止累积误差，增加微分增益提供阻尼。

### 修改2：反转Roll和Pitch控制方向

**文件：** `components/core/crazyflie/modules/src/controller_pid.c`

**修改内容：**
```c
attitudeControllerGetActuatorOutput(&control->roll,
                                    &control->pitch,
                                    &control->yaw);

// 添加控制方向反转
control->roll = -control->roll;   // 反转Roll控制方向
control->pitch = -control->pitch; // 反转Pitch控制方向
control->yaw = -control->yaw;     // 已有的Yaw反转
```

**原理：** 
- BMI088传感器坐标系与Crazyflie电机框架的Roll/Pitch轴方向相反
- 反转控制输出使负反馈回路正确工作
- 防止正反馈导致的姿态发散

### 修改3：修正X模式的RAW计算显示

**文件：** `components/core/crazyflie/modules/src/power_distribution_stock.c`

**修改内容：**
```c
#ifdef QUAD_FORMATION_X
  int16_t r = control->roll / 2;
  int16_t p = control->pitch / 2;
  int32_t raw_m1 = (int32_t)(control->thrust - r + p + control->yaw);
  int32_t raw_m2 = (int32_t)(control->thrust - r - p - control->yaw);
  int32_t raw_m3 = (int32_t)(control->thrust + r - p + control->yaw);
  int32_t raw_m4 = (int32_t)(control->thrust + r + p - control->yaw);
#else
  // NORMAL模式公式
#endif
```

## 验证结果

### 修改前：
```
[PWR] M1:0 M2:0 M3:24205 M4:10319 | T:2091 R:-30287 P:-10963 Y:-1773
[PID] RateActual: R:-2.0 P:0.2 Y:-0.7 | Output: R:-30287 P:-10963 Y:-1773
```
- M1/M2完全无输出
- Roll/Pitch输出饱和在±30000
- 姿态持续发散

### 修改后：
```
[PWR] M1:2435 M2:2807 M3:957 M4:2165 | T:2091 R:-1061 P:419 Y:-395
[PID] RateActual: R:1.7 P:-0.5 Y:0.2 | Output: R:-1061 P:419 Y:-395
```
- 四个电机输出正常 (957-2807范围)
- Roll/Pitch输出在合理范围 (±1100)
- 姿态收敛，不再发散

## 技术总结

### 问题根源
1. **PID参数不匹配硬件特性** - 原始Crazyflie参数针对不同的电机和传感器
2. **传感器坐标系映射错误** - BMI088轴向与代码假设的方向相反
3. **正反馈回路** - 错误的控制方向导致姿态发散而非收敛

### 关键诊断技术
- **分层调试输出** - 从电机输出→控制值→PID输出→传感器读数
- **闭环响应分析** - 观察控制输出与姿态变化的关系判断反馈极性
- **数值范围验证** - 对比理论值与实际值发现异常

### 控制系统设计要点
- **PID调参原则** - 从低增益开始，逐步增加直到出现震荡
- **传感器校准** - 验证坐标系与控制框架的映射关系
- **负反馈验证** - 确保扰动会被控制器纠正而非放大

## 后续优化建议

### 1. 微调PID参数
当前参数可实现基本稳定，但可进一步优化：
- 外环Attitude PID (KP=5.9) 可降至3.0-4.0减少超调
- 在实际飞行中根据响应速度调整

### 2. 传感器校准
- 执行加速度计6面校准
- 陀螺仪零偏校准（已实现）
- 磁力计校准（如使用）

### 3. 系统标定
- 电机输出线性度测试
- 确认电机旋转方向正确性
- 螺旋桨安装方向验证

### 4. 安全测试流程
1. 固定在测试台架
2. 低油门测试电机响应
3. 手动倾斜验证反馈方向
4. 逐步增加油门测试
5. 短时间悬停测试

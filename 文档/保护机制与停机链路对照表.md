# 保护机制与停机链路对照表

> 目的：把当前固件中“会导致电机停转、限制推力、或触发急停”的路径统一整理，便于飞测与排障。
> 
> 适用版本：当前 `diy-drone` 工作区代码（ESP-Drone/Crazyflie 架构改造版）。

---

## 1. 总览对照表

| 编号 | 机制/链路 | 触发条件 | 执行动作 | 电机结果 | 恢复方式 | 关键参数/阈值 |
|---|---|---|---|---|---|---|
| S1 | **Stabilizer 总闸** | `emergencyStop == true` 或 `!vehicleIsArmed()` | 主循环直接走 `powerStop()`，不再做 `powerDistribution()` | 四电机比率置 0 | 清除急停并重新解锁 | `emergencyStop`、`vehicle.isArmed` |
| S2 | **SITAW 翻覆检测急停** | 认为“在飞行中”且连续检测到翻覆特征（基于 `acc.z` 触发器） | `stabilizerSetEmergencyStop()` | 进入 S1 路径后停机 | `vehicleResetEmergency()`/重置急停后重新解锁 | `SITAW_TU_ACC_THRESHOLD`、`SITAW_TU_ACC_TRIGGER_COUNT`、`SITAW_TU_IN_FLIGHT_THRESHOLD` |
| S3 | **RC 丢链强制上锁** | SBUS `failsafe/frameLost` 或超窗失效比率超阈值 | `vehicleDisarm(true)` + `thrust=0` + `vehicleSetRcConnected(false)` | 由于未解锁，S1 生效停机 | RC 恢复后手动重新解锁 | `SIGNAL_LOST_THRESHOLD`、窗口统计参数 |
| S4 | **Commander 看门狗超时** | 控制源超时（分阶段） | 先进入稳姿保守输出，再超时进入 `nullSetpoint` | 推力逐步收敛到 0（最终停机） | 恢复有效控制源输入 | `COMMANDER_WDT_TIMEOUT_STABILIZE`、`COMMANDER_WDT_TIMEOUT_SHUTDOWN` |
| S5 | **远程急停命令** | 收到 `CTRL_CMD_EMERGENCY` | `vehicleEmergencyStop()` + 下发 0 推力 setpoint | 进入 S1 路径后停机 | 手动重置急停并重新解锁 | 网络命令通道 |
| S6 | **车辆状态急停** | 任意调用 `vehicleEmergencyStop()` | `isEmergency=true`、`isArmed=false`、`stabilizerSetEmergencyStop()` | 进入 S1 路径后停机 | `vehicleResetEmergency()` 后重解锁 | `ARMING_STATE_STANDBY_ERROR` |
| S7 | **断言保护停机** | `ASSERT` 失败 | `powerStop()` 后死循环 | 立即停机 | 复位/重启并定位断言 | N/A |
| S8 | **PID 全零联锁（新增）** | `thrust >= deadzone` 且姿态/角速率 PID 全为 0 | 强制 `thrust/roll/pitch/yaw = 0`，并重置 PID | 推力输出被硬拦截 | 恢复有效 PID 参数 | `THRUST_DEAD_ZONE` |
| S9 | **低油门死区清零** | `thrust < THRUST_DEAD_ZONE` | 控制量清零并重置 PID | 低油门不驱动 | 油门拉高 | `THRUST_DEAD_ZONE=3000` |
| S10 | **电机测试任务急停** | `motor_test.estop` 被置位 | motor_test 任务内对所有电机输出 0，并退出测试模式 | 测试通道停机 | 清除 `estop` 后重进测试 | `motor_test.estop` |

## 2. 逐项代码定位（便于快速跳转）

### S1 Stabilizer 总闸
- `if (emergencyStop || !vehicleIsArmed()) { powerStop(); }`
- 文件：`components/core/crazyflie/modules/src/stabilizer.c`

### S2 SITAW 翻覆检测急停
- 编译开关：`SITAW_TU_ENABLED`
- 飞行判定：`sumRatio > SITAW_TU_IN_FLIGHT_THRESHOLD`
- 触发后：`stabilizerSetEmergencyStop();`
- 文件：
  - `components/core/crazyflie/modules/interface/sitaw.h`
  - `components/core/crazyflie/modules/src/sitaw.c`

### S3 RC 丢链强制上锁
- 信号丢失时：`vehicleDisarm(true)`、`extrxSetpoint.thrust = 0`
- 文件：`components/core/crazyflie/modules/src/extrx.c`

### S4 Commander 看门狗超时
- 阶段阈值：
  - `COMMANDER_WDT_TIMEOUT_STABILIZE`（500ms）
  - `COMMANDER_WDT_TIMEOUT_SHUTDOWN`（2000ms）
- 文件：
  - `components/core/crazyflie/modules/interface/commander.h`
  - `components/core/crazyflie/modules/src/commander.c`

### S5 远程急停命令
- `CTRL_CMD_EMERGENCY` -> `vehicleEmergencyStop()`
- 文件：`components/core/crazyflie/modules/src/remote_server.c`

### S6 车辆状态机急停
- `vehicleEmergencyStop()` 将 `isEmergency=true` 且 `isArmed=false`
- 文件：`components/core/crazyflie/modules/src/vehicle_state.c`

### S7 断言保护停机
- `assertFail()` 内调用 `powerStop()`
- 文件：`components/core/crazyflie/utils/src/cfassert.c`

### S8 PID 全零联锁（新增）
- 条件命中后强制清零控制并重置 PID
- 文件：`components/core/crazyflie/modules/src/controller_pid.c`

### S9 低油门死区
- `< THRUST_DEAD_ZONE` 时清零控制量
- 文件：`components/core/crazyflie/modules/src/controller_pid.c`

### S10 电机测试急停
- `motor_test.estop` 置位后立即全电机输出 0
- 文件：`components/core/crazyflie/modules/src/motor_test.c`

## 3. 与“首次 ESC 行程校准”链路的关系

该链路**不是失控保护**，而是维护动作：

1. 地面站写参数：`escCalib.request = 1`
2. 固件在电机任务里检测到请求后置位 RTC 标记并重启
3. 重启后 PWM 初始化完成即执行校准序列（先最大后最小）

相关文件：
- `components/drivers/general/motors/motors.c`
- `apptest/src/DroneServer.js`
- `apptest/public/js/motorTest.js`

## 4. 飞测建议（最小验证清单）

1. **S2 翻覆急停验证**：桨拆除后上电，短暂拉起推力并模拟翻覆，观察是否触发急停。
2. **S3 丢链保护验证**：解锁后断开发射端，确认自动上锁与推力归零。
3. **S4 控制源超时验证**：停止发送控制指令，观察 500ms/2000ms 两阶段行为。
4. **S8 全零 PID 联锁验证**：将姿态与角速率 PID 全置 0，确认推力被硬拦截。
5. **S10 测试急停验证**：电机测试模式下触发 `motor_test.estop`，确认立即停机。

> 安全注意：所有验证先“拆桨”进行；确认日志与参数状态一致后再做带桨测试。

## 5. 常见误判说明

- 网页端 `motorOutputs` 可能来自“遥测字段”或“后端缓存回填”，不一定等价于最终 PWM。
- 判定真实停机应同时看：
  1) `vehicle.isArmed`、`vehicle.failsafe`；
  2) 固件日志（如 `RC SIGNAL LOST`、`EMERGENCY STOP`）；
  3) 实际电机声音/转速。

## 6. 前端“失控保护”当前可显示项（按优先级）

地面站保护状态由 `apptest/public/js/dashboard.js` 中 `evaluateProtectionState()` 决定，当前顺序如下：

1. **翻覆检测急停（SITAW）**
   - 条件：`isTumbled == true`
   - 显示原因：`翻覆检测触发急停(SITAW_TU)`
   - 措施：`已切断电机输出，需排除风险后复位解锁`

2. **紧急停机（E-STOP）**
   - 条件：`isEmergency == true`
   - 显示原因：`紧急停机(E-STOP)已触发`
   - 措施：`已切断电机输出并强制上锁`

3. **RC 丢链保护**
   - 条件：`failsafeState == 3`
   - 显示原因：`故障安全: 遥控信号丢失`
   - 措施：`强制上锁并将推力置零`

4. **严重低电（若飞控上报）**
   - 条件：`failsafeState == 2`
   - 显示原因：`故障安全: 严重低电`
   - 措施：`进入严重低电保护，建议立即降落`

5. **其他故障安全状态**
   - 条件：`failsafeState > 0` 且不满足以上特例
   - 显示原因：`故障安全: <映射名称>`
   - 措施：`已进入保护状态，请尽快处置`

6. **解锁状态异常**
   - 条件：`armingState == 3`（`STANDBY_ERROR`）
   - 显示原因：`解锁状态异常(STANDBY_ERROR)`
   - 措施：`禁止解锁，需排除故障后复位`

7. **已解锁但无控制源**
   - 条件：`isArmed == true && controlSource == 0`
   - 显示原因：`已解锁但无有效控制源`
   - 措施：`等待控制链路恢复，超时将触发看门狗保护`

8. **正常**
   - 条件：以上都不满足
   - 显示：`正常 / 监控中`

## 7. 翻覆急停显示链路（已打通）

为避免“电机已归零但前端不知道是翻覆急停”的误判，已新增翻覆状态位透传：

1. **固件遥测状态位新增**
   - `STATUS_FLAG_TUMBLED = (1 << 6)`
   - 文件：`components/core/crazyflie/modules/interface/remote_server.h`

2. **固件打包遥测时写入翻覆状态**
   - 条件：`sitAwTuDetected()` 为真时置位 `STATUS_FLAG_TUMBLED`
   - 文件：`components/core/crazyflie/modules/src/remote_server.c`

3. **Node 协议解析层识别该状态位**
   - `STATUS_FLAGS.TUMBLED = 0x40`
   - `telemetry.isTumbled = !!(statusFlags & STATUS_FLAGS.TUMBLED)`
   - 文件：
     - `apptest/src/protocol.js`
     - `apptest/src/parser.js`

4. **前端优先显示翻覆急停原因**
   - 文件：`apptest/public/js/dashboard.js`

> 注意：该显示链路生效前提是“固件与地面站同时更新到同一版本”。

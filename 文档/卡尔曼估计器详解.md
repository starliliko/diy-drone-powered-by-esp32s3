# 卡尔曼估计器详解

本文档详细介绍 ESP-Drone 项目中基于 Crazyflie 的扩展卡尔曼滤波器（EKF）的工作原理、数据流和实现细节。

## 1. 概述

卡尔曼估计器是无人机状态估计的核心组件，负责融合多种传感器数据，输出准确的位置、速度和姿态信息。

### 核心文件

| 文件                 | 功能                   |
| -------------------- | ---------------------- |
| `kalman_core.c`      | 卡尔曼滤波核心算法     |
| `estimator_kalman.c` | 卡尔曼估计器任务与接口 |
| `estimator.c`        | 估计器抽象层           |
| `stabilizer_types.h` | 数据类型定义           |

## 2. 状态向量

### 2.1 内部状态 S[9]

卡尔曼滤波器维护一个 9 维误差状态向量：

| 索引 | 状态名        | 符号 | 含义                  | 单位 | 坐标系   |
| ---- | ------------- | ---- | --------------------- | ---- | -------- |
| 0    | `KC_STATE_X`  | x    | X 位置                | m    | 世界     |
| 1    | `KC_STATE_Y`  | y    | Y 位置                | m    | 世界     |
| 2    | `KC_STATE_Z`  | z    | Z 位置（高度）        | m    | 世界     |
| 3    | `KC_STATE_PX` | vx   | X 速度                | m/s  | **机体** |
| 4    | `KC_STATE_PY` | vy   | Y 速度                | m/s  | **机体** |
| 5    | `KC_STATE_PZ` | vz   | Z 速度                | m/s  | **机体** |
| 6    | `KC_STATE_D0` | δθ₀  | 姿态误差 (roll 方向)  | rad  | -        |
| 7    | `KC_STATE_D1` | δθ₁  | 姿态误差 (pitch 方向) | rad  | -        |
| 8    | `KC_STATE_D2` | δθ₂  | 姿态误差 (yaw 方向)   | rad  | -        |

### 2.2 附加状态

除了误差状态向量，还维护：

```c
typedef struct {
    float S[KC_STATE_DIM];      // 误差状态向量 (9维)
    float P[KC_STATE_DIM][KC_STATE_DIM];  // 协方差矩阵 (9×9)
    float q[4];                 // 姿态四元数 [w, x, y, z]
    float R[3][3];              // 旋转矩阵 (机体→世界)
    float baroReferenceHeight;  // 气压计参考高度
} kalmanCoreData_t;
```

## 3. 输入传感器

### 3.1 传感器数据类型

| 传感器       | 数据结构            | 主要字段             | 更新频率 |
| ------------ | ------------------- | -------------------- | -------- |
| IMU 加速度计 | `Axis3f acc`        | x, y, z (G)          | 500Hz    |
| IMU 陀螺仪   | `Axis3f gyro`       | x, y, z (°/s)        | 500Hz    |
| 气压计       | `baro_t`            | asl (m)              | 25Hz     |
| ToF 测距     | `tofMeasurement_t`  | distance (m), stdDev | 100Hz    |
| 光流         | `flowMeasurement_t` | dpixelx, dpixely, dt | 100Hz    |

### 3.2 光流数据格式

```c
typedef struct {
    uint32_t timestamp;   // 时间戳 (微秒)
    float dpixelx;        // X方向累积像素位移
    float dpixely;        // Y方向累积像素位移  
    float stdDevX;        // X 标准差
    float stdDevY;        // Y 标准差
    float dt;             // 采样间隔 (秒)
} flowMeasurement_t;
```

### 3.3 ToF 数据格式

```c
typedef struct {
    uint32_t timestamp;   // 时间戳 (微秒)
    float distance;       // 距离 (米)
    float stdDev;         // 标准差
} tofMeasurement_t;
```

## 4. 输出状态

### 4.1 state_t 结构

```c
typedef struct {
    attitude_t attitude;           // 欧拉角 (roll, pitch, yaw) [度]
    quaternion_t attitudeQuaternion; // 四元数 [w, x, y, z]
    point_t position;              // 位置 [m] (世界坐标系)
    velocity_t velocity;           // 速度 [m/s] (世界坐标系)
    acc_t acc;                     // 加速度 [G] (世界坐标系)
} state_t;
```

### 4.2 坐标系转换

**注意**：内部速度 `S[PX], S[PY], S[PZ]` 是**机体坐标系**，输出前需转换：

```c
// 机体坐标系 → 世界坐标系
velocity.x = R[0][0]*S[PX] + R[0][1]*S[PY] + R[0][2]*S[PZ];
velocity.y = R[1][0]*S[PX] + R[1][1]*S[PY] + R[1][2]*S[PZ];
velocity.z = R[2][0]*S[PX] + R[2][1]*S[PY] + R[2][2]*S[PZ];
```

### 4.3 欧拉角计算

从四元数 q = [w, x, y, z] 计算：

$$\text{yaw} = \arctan2(2(q_xq_y + q_wq_z), q_w^2 + q_x^2 - q_y^2 - q_z^2)$$

$$\text{pitch} = \arcsin(-2(q_xq_z - q_wq_y))$$

$$\text{roll} = \arctan2(2(q_yq_z + q_wq_x), q_w^2 - q_x^2 - q_y^2 + q_z^2)$$

## 6. 预测步骤详解

### 6.1 位置预测

位置在世界坐标系中更新，需要将机体速度通过旋转矩阵转换：

$$\vec{x}_{new} = \vec{x}_{old} + R \cdot \vec{v}_{body} \cdot dt + \frac{1}{2} R \cdot \vec{a}_{body} \cdot dt^2 - \frac{1}{2}\vec{g} \cdot dt^2$$

其中：
- $R$ 是 3×3 旋转矩阵（机体→世界）
- $\vec{v}_{body} = [S_{PX}, S_{PY}, S_{PZ}]^T$
- $\vec{g} = [0, 0, 9.81]^T$ m/s²

### 6.2 速度预测

速度在机体坐标系中更新：

$$\vec{v}_{new} = \vec{v}_{old} + dt \cdot (\vec{a}_{body} + \vec{\omega} \times \vec{v}_{old} - R^T \cdot \vec{g})$$

其中：
- $\vec{a}_{body}$ 是加速度计测量值
- $\vec{\omega}$ 是陀螺仪测量的角速度
- $\vec{\omega} \times \vec{v}$ 是科里奥利力补偿

### 6.3 姿态预测

使用四元数积分陀螺仪数据：

$$q_{new} = q_{\Delta} \otimes q_{old}$$

其中增量四元数：
$$q_{\Delta} = \begin{bmatrix} \cos(\frac{|\vec{\omega}|dt}{2}) \\ \frac{\vec{\omega}}{|\vec{\omega}|} \sin(\frac{|\vec{\omega}|dt}{2}) \end{bmatrix}$$

## 7. 测量更新详解

### 7.1 标量更新算法

所有传感器更新都使用标量更新公式（`scalarUpdate`）：

```
输入: H (测量雅可比), error (测量误差), R (测量噪声方差)

1. 计算卡尔曼增益:
   K = P * H' / (H * P * H' + R)

2. 更新状态:
   S = S + K * error

3. 更新协方差:
   P = (I - K*H) * P * (I - K*H)' + K * R * K'
```

### 7.2 ToF 测距更新

**测量方程**：
$$h_{tof} = \frac{S_Z}{R_{2,2}} = \frac{z}{\cos(\theta)}$$

其中 $R_{2,2}$ 是旋转矩阵的 (2,2) 元素，等于 $\cos(\text{倾斜角})$。

**更新过程**：
```c
predictedDistance = S[KC_STATE_Z] / R[2][2];
measuredDistance = tof->distance;  // 单位: 米
error = measuredDistance - predictedDistance;
H[KC_STATE_Z] = 1 / R[2][2];
scalarUpdate(H, error, tof->stdDev);
```

**更新的状态**: `S[KC_STATE_Z]` (高度)

### 7.3 光流更新

**相机模型参数**：
- $N_{pix} = 30$ 像素（传感器分辨率）
- $\theta_{pix} = 4.2°$ 每像素视场角

**测量方程**（X方向）：
$$\text{predictedNX} = \frac{dt \cdot N_{pix}}{\theta_{pix}} \cdot \left( \frac{v_x \cdot R_{2,2}}{z} - 1.25 \cdot \omega_y \right)$$

其中：
- $v_x = S_{PX}$ 机体X速度
- $z = S_Z$ 高度
- $\omega_y$ 机体Y轴角速度（陀螺仪）
- $1.25$ 是陀螺仪补偿因子

**测量值**：
$$\text{measuredNX} = \text{flow.dpixelx}$$

**雅可比矩阵**：
$$H_Z = \frac{N_{pix} \cdot dt}{\theta_{pix}} \cdot \frac{R_{2,2} \cdot v_x}{-z^2}$$
$$H_{PX} = \frac{N_{pix} \cdot dt}{\theta_{pix}} \cdot \frac{R_{2,2}}{z}$$

**更新的状态**: `S[KC_STATE_Z]`, `S[KC_STATE_PX]`, `S[KC_STATE_PY]`

## 8. MTF01 光流数据转换

### 8.1 MTF01 输出格式

MTF01 传感器输出：
- `flow_vel_x/y`: 光流速度，单位 cm/s @ 1m 高度
- 实际速度 = flow_vel × height(m)

### 8.2 转换公式推导

**卡尔曼期望的 dpixel 格式**（从测量方程反推）：
$$dpixel = \frac{dt \cdot N_{pix}}{\theta_{pix}} \cdot \frac{v_{actual}}{h}$$

**MTF01 的归一化特性**：
$$v_{actual} = \frac{flow\_vel \cdot h}{100}$$
（其中 /100 是 cm/s → m/s 转换）

**代入得**：
$$dpixel = \frac{dt \cdot N_{pix}}{\theta_{pix}} \cdot \frac{flow\_vel \cdot h}{100 \cdot h} = \frac{dt \cdot N_{pix}}{\theta_{pix}} \cdot \frac{flow\_vel}{100}$$

**关键发现**：高度 $h$ 被消掉了！

### 8.3 最终转换代码

```c
// 相机常数（与 kalman_core.c 一致）
#define FLOW_NPIX        30.0f           // [pixels]
#define FLOW_THETAPIX    (DEG_TO_RAD * 4.2f)  // [rad]

// 转换公式
float dt = 1.0f / MTF01_TASK_FREQ;  // 0.01s @ 100Hz
float flowScale = dt * FLOW_NPIX / FLOW_THETAPIX / 100.0f;

flowData.dpixelx = (float)flowVelX * flowScale;
flowData.dpixely = (float)flowVelY * flowScale;
```

**数值计算**：
- $flowScale = 0.01 \times 30 / 0.0733 / 100 \approx 0.0409$

### 8.4 为什么这样转换？

| 方面           | 说明                                                      |
| -------------- | --------------------------------------------------------- |
| **陀螺补偿**   | 卡尔曼内置陀螺仪补偿 ($-1.25 \cdot \omega$)，无需外部处理 |
| **高度无关**   | MTF01 归一化输出使得转换公式与高度无关，消除测距误差影响  |
| **传感器兼容** | 统一 dpixel 接口，兼容 PMW3901 等其他光流传感器           |

## 9. 状态收敛 (Finalize)

### 9.1 姿态误差融合

每个周期结束时，将姿态误差 `D0, D1, D2` 融合到四元数中：

```c
// 误差向量转增量四元数
float angle = sqrt(D0² + D1² + D2²);
float dq[4] = {cos(angle/2), sin(angle/2)*D0/angle, ...};

// 四元数更新
q_new = dq ⊗ q_old;
normalize(q_new);

// 更新旋转矩阵
R = quatToRotMat(q);

// 清零误差状态
D0 = D1 = D2 = 0;
```

### 9.2 协方差旋转

姿态融合后，协方差矩阵也需要相应旋转，保持一致性。

## 11. 常见问题

### Q1: 为什么速度是机体坐标系？

机体坐标系速度更便于控制：
- 直接对应推力方向
- 避免姿态耦合
- 简化控制器设计

### Q2: 光流更新为什么需要陀螺仪？

无人机旋转会在光流中产生"伪速度"，必须通过陀螺仪数据补偿。这就是为什么推荐使用 dpixel 格式而非直接速度。

### Q3: 高度估计不准怎么办？

检查项：
1. ToF 传感器安装是否垂直向下
2. `tofStdDev` 参数是否合适
3. 是否在 ToF 有效量程内（2-2000mm）

### Q4: 位置漂移怎么处理？

光流本质是速度传感器，长期积分会漂移。解决方案：
1. 确保光流质量指标 > 0
2. 增大 `flowStdDev` 减少光流权重
3. 使用绝对定位（如 UWB）校正
